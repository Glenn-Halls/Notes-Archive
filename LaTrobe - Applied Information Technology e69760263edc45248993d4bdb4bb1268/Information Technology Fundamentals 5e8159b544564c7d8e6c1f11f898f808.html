Week 2: Computer Hardware

  • Transistors - Boolean Algebra - transistors forming logic gates - combining to form mathematical functions - Integrated Circuits
  • RAID - Redundant Array of Inexpensive Disks
    • RAID 0 - Striping
      • Data is striped across all disks improving I/O performance and utilising all capacity however failure of one drive = failure of all.
      • Minimum 2 drives
    • RAID 1 - Mirroring
      • Mirrors Data across multiple drives making read slower but write faster. Failure of one drive can be accomodated
      • Minimum 2 drives
    • RAID 5 - Distributed
      • Distributes data across all drives including parity information which can be used to reconstruct a drive if needed.
      • Minimum 3 drives, only 1 drive of capacity is lost.
    • RAID 10 - Hybrid 1 & 0
      • Combines performance of RAID 1 and reliability of RAID 0 but is the most expensive
      • Minimum 4 drives
  • Computer System: Input —> Processor / Memory —> Output
  • Virtualisation - Multiple Virtual Systems on one system
  • Motherboard Form Factor - Extended Vs Standard Vs Micro ATX
  • Memory Requires continuous electric charge
  • Storage retains data without electricity
  • Non Volatile Memory Express - NVMe

      • Macine Language - does not use a compiler
      • Low-level languages with fast processing time

Week 10: Data Structures and Algorithms

  • A data structure is a way to store, organise and manage information in a way that allows you to access or modify their values
  • Data Structures:
    • Linear: elements are stored and retrieved in a sequential manner. This includes stacks, queues, arrays, linked lists
      • Stacks: an ordered list where data insertion and deletion can only occur at one end, called the top. Consider the ‘last in, first out’ (LIFO) and ‘first in, last out’ (FILO) principles
        • Push: insertion operation on a stack is called a ‘push’
        • Pop: deletion operation removing the top element
      • Queue: another ordered list structure but in this structure insertion and deletion occurs at opposite ends of the list. Consider ‘first in, first out’ (FIFO) and ‘last in last out’ (LILO) principles
        • Enqueue: insertion operation on a queue (rear)
        • Dequeue: deletion operation on a queue (front)
      • Arrays: these data structures are used to store elements of the same type together in a sequential memory location where any element can be accessed directly via its index
        • Index from 0 - (n-1)
        • Traversing an array: this refers to accessing each element in the array, one at a time, in sequential order. This can be used to update or output values, count elements or even perform operations on the elements
        • Inserting an array element: can be done at any index of an array, including the beginning or end. Inserting at index x will insert at that index and move all elements ≥ x to x+1
        • Deleting an array element can be done at any index and similarly to inserting, will change the index of all elements > x to x-1
      • Linked Lists: used to store elements of the same type together where one node containing list elements is linked to another node containing list elements via a memory pointer
        • A node is made up of two parts: the value of the element (data) and the pointer (link)
        • The first node of the list is the head, and contains no value where the last node of the list points to NULL
        • Traversing the list - similar to traversing an array, this can be used to count, search, perform operations and update elements. The current ‘index’ of the list is referred to as the “currentNode”
        • Insertion in the list - similar to an array but no update to other nodes needed as there is no index. If insertion occurs at the beginning, the head will point to the new node
        • Deleting an element from the list - similar to an array but the nodes connected to the deleted element will need to be updated to ensure continuity of the string. Again, consider the head if the first element is deleted
      • Linked lists Vs Arrays:
        • Size: LL can grow and shrink, A must pre-allocate
        • Memory Allocation: LL > A
        • Operations: Updating an element much faster in an A, insertion and deletion much faster in LL as do not have to update other elements
    • Non-Linear: data structures are arranged in a hierarchical manner where one element may be connected to one or more elements. These may be either graph or tree-based data structures. Examples include Binary & Red-black tree structures
      • Trees:
        • Non-linear, hierarchical, nodes connected to one or more nodes
        • Root: the starting point of a tree, has no parent
        • Subtree: a part of the tree containing a node and its descdendants
        • Parent node / child node
        • Edge = branch = link from parent node to child node
        • Leaf node: a child node with no children
        • Depth of node = number of edges from child to root
        • Height of tree = maximum depth among all leaf nodes
        • Binary trees: can have ≤ 2 branches and are the basic data structures of data science
          • Full binary trees: all nodes have 0 or 2 child nodes
          • Complete binary trees: all nodes except last level have children
          • Perfect binary trees: all nodes have 2 children and all leaf nodes are the same depth
        • Binary Search Tree: an ordered node binary tree data structure. Must have the following properties:
          • Left subtree key values < root key
          • Right subtree key values > root key
          • All subtrees are binary search trees
          • Skewed tree = all nodes have one or no children
          • Height Balancing takes a skewed node and balances so the height difference between left and right is either 0 or 1
        • Red-black trees: one of the primary search structures for Java and C++, every node has a color, either red or black. This ensures the height of the binary tree is not skewed.
          • The root node must be black and the children of red nodes must be black
    • Algorithm: from ‘algorism’ (computing using Arabic numbers) and ‘arithmos’ (a Persian mathematician). Programs implement algorithms to solve problems
      • A set of well-defined, finite steps to solve a repeatable problem.
      • Types of algorithms:
        • Recursive: calls itself repeatedly until the problem is solved
        • Brute-force: tries all possibilities until one succeeds
        • Sorting: such as selection, sort, bubble sort, quick sort
        • Divide and conquer: splits the problem into smaller sub-problems, the solutions to which are combined to produce a solution to the whole
      • Comprise of:
        • Input, output, computation, selection, iteration (looping)
      • Expressed by:
        • Natural language, flowcharts, pseudocode, programming language
      • Analysing Algorithms:
        • Efficiency - memory use & processing power
        • Growth of function = how much running time increases as the size of the problem increases
        • User-friendliness, security, maintenance, space
        • Asymptomatic Analysis
          • Analyses an algorithm with various input sizes and measures performance in a worst case, best case and average case scenario
          • Big-O notation:
            • https://www.linkedin.com/pulse/big-o-notation-simple-explanation-examples-pamela-lovett/
      • Designing Algorithms:
        • Greedy method: work is performed in stages with each stage caring only about itself. Relies on the principle that local optimisation —> global optimisation
        • Divide and conquer: divide into subproblems, solve them and then recombine
        • Dynamic programming: similar to D&C but stores output of subproblems into data structures to be reused again.
        • Reduction / Transform and conquer: turn a new problem into a previously known problem for which there is already an answer

Week 11: Cloud Computing

  • AWS origin story - black Friday capacity —> PAYG computing —> Netflix
  • The branch of cloud computing dealing with software is called SaaS —> Software as a Service
  • Grid computing and the sharing of computational resources across organisations
  • 5 Characteristics of cloud computing:
    • Ubiquitous
    • On-demand self-service
    • Broad network access
    • Resource pooling (and smaller size provisioning)
      • Multi tenant pooling (selling more resources than what you have available)
    • Rapid elasticity and measured service
  • 3 Cloud Service Models:
    • Infrastructure:
      • IaaS - Infrastructure as a Service
      • Lowest level of service on offer by the cloud
      • Highest level of user control
      • Maintained by Vendor: virtualisation, servers, storage, networking
      • Maintained by User: O/S, Middleware, Runtime, Data, Applications
      • Consider: compute resources, virtual network layer, load balancing, storage
    • Platform:
      • PaaS - Platform as a Service
      • Offers the infrastructure of IaaS plus the platform level services
      • Example includes the Microsoft Azure service
    • Software:
      • SaaS - Software as a Service
      • Highest level of service that can be provided
      • Cliniko, Google Docs, CRM - Customer Relationship Management
  • 4 Cloud Deployment Models:
    • Private Cloud:
      • Infrastructure is provisioned for exclusive use by a single organisation or cusromer.
      • By far the most expensive model
    • Community Cloud:
      • Resources are owned, operated and managed by the organisations in the community or a third party on their behalf.
      • Consider the shared compute resources across research labs in a Uni
    • Public Cloud:
      • Anyone from the general public can access these resources - by far the most popular model
      • Resources exist on the premises of the cloud provider
    • Hybrid Cloud:
      • Combines one or more of the above models
      • Often, this is done in an attempt to reduce the impact of ‘cloud bursting’, where there is a run on cloud resources as in this case, load balancing between cloud resources may take place
  • 3 Main Providers:
    • Amazon Web Services (AWS)
      • EC2 - Elastic Compute Cloud: virtual computing environment offering templated AMIs (Amazon Machine Images) that can be deployed in seconds. While there are hundreds of AMIs templated, users can also create their own
      • S3 - Simple Storage Service: can be independent or part of a machine, used for pure storage but not for databases. Often used for backup
      • ELB - Elastic Load Balancer: automatically creates and destroys VMs in a manner dictated by the user
      • RDS - Relational Database Service: can host SQL servers, requires a VM
    • Google Cloud Platform (GCP)
    • Microsoft (Azure)
  • Data Centres = collection of clusters = collection of servers
