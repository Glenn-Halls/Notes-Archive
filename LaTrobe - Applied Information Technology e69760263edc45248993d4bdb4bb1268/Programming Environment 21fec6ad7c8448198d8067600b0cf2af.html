Week 1: Introduction to Computer Programming

  • Machine Languages (binary digits) Vs Assembly / Low-Level Languages (tied to a computer’s architecture - use an assembler to translate to machine language) Vs High Level Languages (to make code more readable, less error prone and faster to develop, utilises a compiler).
  • Programming Development Life Cycle:
    • Problem Definition
      • Understand the problem and the inputs, outputs, constraints and requirements
    • Problem Analysis
      • Including breaking down into sub-problems
    • Algorithm Design
      • Outlining all the steps and actions needed to solve the problem, this is where flowcharts and pseudocode may be used, should also include error handling
    • Code Development
      • Conversion of algorithm to code including comments and documentation
    • Testing and Debugging
      • Logical Vs Syntax errors Vs debugging, all of which should be documented
    • Maintenance —> Problem Definition
      • Including fixing errors, updating, upgrading. Often accounts for 50% of time spent on a program
  • Number Systems:
    • Binary (0 & 1)
      • right-most = least significant bit - LSB
      • left-most = most significant bit - MSB
      • 10110 = 1*2^4 + 0*2^3 + 1*2^2 + 1*2^1 + 0*2^0
      • 10110 = 1*16 + 0 *8 + 1*4 + 1*2 + 0*1
      • 10110 = 16 + 0 + 4 + 2 + 0
      • 10110 = 22
        • To Convert Back:
          • 22 / 2 / 2 / 2 / 2 / 2 with remainders:
            • 0 / 1 / 1 / 0 / 1
            • reverse this to 10110
    • Decimal (0-9)
    • Octal (0-7)
    • Hexadecimal (0-9, A-F)
      • Conversion tables - decimal to binary, etc
      • Conversion tables, hex to binary, etc
  • Flowcharts:
    • Title —> start and end —> list all steps —> arrange sequentially —> connect —> review and refine
  • Pseudocode - representing algorithms in English-like statements:
    • Statements in English and programming language neutral
    • Each instruction on a separate line
    • Keywords, verbs, control statements with leading capital
    • Indentation of loops & sections with 2-4 spaces
    • Written top-bottom, one entry, one exit
    • Groups of statements may be modules with group names
    • Verbage:
      • Read / Accept / Get
        • Read for media, accept / get for device (keyboard) input
      • Print / Write / Display / Prompt
      • Compute: + / - / * / / / ( ) / =
        • Accumulate / Increment / Decrement
      • Initialise to initiate a variable, let to assign a value (Let var = 1)
      • If —> Then —> Else —> EndIf
      • While —> EndWhile
    • For this course:
      • line numbers
      • start with Start, end with End, use described keywords
  • The importance of “Desk-Checking” Pseudocode:
    • Test Data: comprehensive, tests all paths, tests error situations, tests boundary conditions

Week 3: Java Language Fundamentals

  • Variables abstract the use of memory in a programming language in order to take care of the complications of performing actual memory management.
  • Variables:
    • Are unique and have three attributes: name, type and value
    • Can start with a letter (upper or lower), or an underscore
      • Subsequent characters can be letters, numbers, or underscores. By convention, $ is not used, however it can be used in some situations
    • Name is case sensitive, and can be any length but whitespace is not allowed
    • Some names are keywords and thus reserved by Java:
    • Compound names should use camelCase
    • If a variable stores a constant value (final), it should be written in only capitals, such as PI
    • Names SHOULD always start with a lowercase letter OR an underscore
  • There are 8 primitive data types in Java:
    • byte (8 bits: -128 to 127)
    • short (16 bits: -32,768 to 32,767)
    • int (32 bits: ±2.147×109\pm2.147\times10^9±2.147×109﻿)
      • can be represented in:
        • octal = 0 prefix
        • hexadecimal = 0x
        • binary = 0b
    • long (64 bits: ±9.223×10(18)\pm9.223\times10^(18)±9.223×10(18)﻿
    • float (with 7 decimal places)
      • must be suffixed with an “f” to NOT be a double
      • can use scientific notation: 1.23e123f
    • double (with 16 decimal places)
      • is the default decimal variable in Java
      • can use scientific notation: 1.23e123
    • boolean (single bit)
    • char (stored as a 16 bit integer)
      • https://www.ascii-code.com/
    • While not technically a primative data type, Strings are often thought of as so due to their integral nature and comprehensive support within Java
  • Before using a variable in Java, it must first be declared
  • Escape characters:
    • \b - backspace
    • \n - newline
    • \r - carriage return
    • \f - form feed
    • \t - horizontal tab
    • \” - double quote
    • \’ - single quote
    • \\ - backslash - very important in windows when a file name is part of a string
    • \0 - null
  • Type casting:
    • Conversion of one object or variable to another:
    • Classified as either:
      • Widening casting (implicit):
        • two types must be compatible with the target type larger than the source:
        • byte —> short —> int —> long —> float —> double
      • Narrowing casting (explicit):
        • must be states or an error will be thrown by the compiler
        • may lose value
        • double D1 = 123.456; long L1 = (long)D1; int I1 = (int)L1
  • Output formatting:
    • System.out.printf("format-string" [, $arg1, $arg2, ...]);
    • preceded with a $
    • [flags]
      • - = left justify
      • + = output a + or - numerical sign
      • [x]0 = forces numerical values to be [x] zero padded
      • , = comma grouping separator
      • = space displays - if negative or a space if positive
    • [width]
      • minimum number of characters to be written to the output
    • [precision]
      • number of digits of precision in floating-point values
    • conversion characters
      • determine which numeric format is to be displayed
      • d = decimal integer
      • [.8]f = floating point number - [8 decimal places]
      • c = character (capital C will capitalise)
      • s = string (capital S will capitalise)
      • h = hashcode
      • n = newline
  • Data dictionary:
    • Made up of 3 columns: Name, Type, Description
  • Operators:
    • Consider using parentheses to clarify long statements
    • Order of Precedence:
    • Arithmetic:
      • +, -, *, /, %
    • Assignment (=)
    • Unary (require only one operand)
      • +, -, !, ++, - -
      • ++ and - can be prefixed or postfixed, changing their declaration timing during expressions
    • Relational:
      • ==, !=, >, >=, <, <=
    • Bitwise:
      • Logical:
        • & - AND
        • | - OR
        • ~ - NOT
        • ^ - XOR
      • Shift:
        • when shifting to the left or right, the extra bits are determined by the sign of the value (+ / -)
        • >> = right shift
        • << = left shift
        • >>> = unsigned right shift
          • values added are 0s
        • can use Integer.toBinaryString()
    • Boolean Logical:
      • && - true && true = true, all other false
      • || - false || false = false, all other true
    • Ternary:
      • ? - takes 3 operands
      • boolean expression ? value1 : value2
  • Expressions yield results (x + y)
  • Statements end with a “;”
    • Expression statements
      • x = 3;
      • System.out.println(”Hello, World”);
    • Declaration statements
      • int i;
      • string name = “John Smith”
      • place declarations at the top of a code block for better readability
    • Statement blocks
      • code blocks, preceded with a class, method or control declaration

Week 7: Arrays

  • Can hold primitive data types
  • Provide fast access to their elements, which are referred to by their index
  • Have a fixed size
  • Exist in a contiguous block of memory
  • Can be one-dimensional or multi-dimensional
  • Declaring an Array:
    • int[] anArray; - encouraged
      • anArray = new int[x]; - initialising
        • size can later be changed, but data type can not.
    • int anArray[]; - discouraged
  • Initialising and declaring an Array:
    • int[] anArray = new int[x];
  • Assigning a value to an element:
    • anArray[e] = x;
  • Declaring, Initialising and Assigning at once:
    • int[] anArray = {a, b, c, d, e};
  • Length attribute is vital for manipulation of arrays, especially in loops:
    • anArray.length
  • Copying an array must be done one element at a time unless the arrayCopy function is used:
    • System.arrayCopy(fromArray, [start-element], toArray, [start-element], [finish-element]);
  • The Arrays class has other useful functionality for arrays:
    • import java.util.Arrays;
    • Arrays.binarySearch(myArray, value);
      • gives a value of -11 if not found
      • this method will only work with sorted arrays - to sort your array, use:
        • Arrays.sort();
  • 2D arrays:
    • int[][] myArray = new int[rows][columns];
    • String[][] grid = {{"a1", "a2", "a3"},{"b1", "b2", "b3"},{"c1", "c2", "c3"}}

Week 8: Methods

  • Non-library methods = ‘user-defined methods’
  • Within a file, the position of the method is irrelevant; it can be placed either above or below the main( )
  • Types of user-defined methods:
  • Void - no parameter passing
    • private static void methodName( )
  • Void - parameter passing
    • public static void methodName(int x, int y)
  • Methods that return values
    • value can be of any data type
    • private static int methodName(int z)
  • The static modifier allows the method to be shared with all objects in its class without creating an object of its class
  • 3 Types of Errors:
    • Compilation Errors
      • will not allow program compilation
    • Run-Time - Errors
      • such as divide by 0
    • Logic Errors
      • runs without failing but produces unexpected result/s
  • Exception handling:
    • Errors from which a program cannot recover Vs Exceptions that halt normal program execution in order to deal with an error by throwing to an Exception Handler which is a block of code that reacts to a specific exception to deal with the problem
    • Predefined Exceptions:
      • ArithmeticException - divide by 0
      • NegativeArraySizeException
      • NullPointerException - attempt to access an object’s method or attribute before the object has been instantiated
      • ArrayIndexOutOfBoundsException
      • NumberFormatException - when a method that converts a string to a number receives a string that it cannot convert
      • Exception - at the very least:
        • catch (Exception e) {
          • System.out.println("Exception " + e)
        • }
    • Try:
      • defines a block of code that may throw exceptions. If this does occur, an optional catch block can deal, with the optional finally block following up regardless
    • Catch:
      • catch (Exception e) {
        • System.out.println(e.getClass())
      • }
      • used to declare a block of code to be executed if an exception occurs during the preceding try block
      • can use
    • Finally:
      • executes a block of statements regardless as to weather an exception was encountered or not
    • Throw and Throws also exists but will not be covered
    • As these checks occur in order, the general exception Exception e should always come last
    • Multicatch:
      • multiple catch statements can be combined with the use of | in between statements in parentheses
      • catch (exception_1 | exception_2 var)

WEEK 9: Object and Data Structure

  • PYTHON
    • Statically typed language vs Python, which is a dynamically-typed language, meaning that variables do not have to be declared. Additionally, they can change type whenever an operation is performed on them
    • does not have the char data type
    • strings can be indexed as you would with an array [x]
    • strings can be sliced into substrings with [a:b]
      • either a or be can be left blank to slice the start or the end of a string
    • substrings can be ‘strided’ (certain characters omitted) with:
      • [a:b:x]
      • x default value is 1 and is standard behaviour
    • strings can also be negatively indexed in order to count from the end of the string
      • string[-a:-b]
    • Useful string functions:
      • len(string)
      • string.isdigit()
      • string.count(x, [start], [end])
      • string.find(x, [start], [end])
        • will return index if found, -1 if not
      • string.upper()
      • string.lower()
      • string.replace(old, new, [count])
      • string.split(identifier, maxsplit)
      • string.format()
        • uses {} as placeholders in the string and replaces these with arguments (as many arguments as there are brace pairs
        • can also use {named} indexes
      • format()
        • within the string, the format header must be contained in curly braces and preceeded by a colon ie {:x}
        • alignment: <, >, =, ^
        • positive or negative declaration: +, -, Space
        • integer presentation formats:
          • b - binary
          • c - unicode
          • d - decimal
          • o - octal
          • x - hex (lowercase letters)
          • X - hex (uppercase letters)
          • n - number
          • formatting decimals:
            • use one of the floating point and decimal presentation types:
              • :e
              • :E
              • :f
              • :F
              • :g
              • :G
              • :%
            • .{} can define the number of decimal points
            • {:.{}f} --> (a, b)
    • Lists are created with square brackets []
      • negative indexing may also be used
      • slicing may also be used
      • len() may also be used
      • .append() to add an element to the end of the list
      • .insert(x, element) to insert an element into position x
      • .remove(value) to remove element of value value
      • pop(e) remove element from position e
        • if no argument, the last item in the list will be removed
      • del list[] to delete an element, similar to pop, but if no argument, the entire list will be deleted, as if it were never initiated
      • clear() to remove all elements from a list without deleting the variable
    • Tuples are immutable lists lists and are created with ( )
      • empty tuples may be created
      • when creating a one element tuple a comma must come after the single element so that python knows it is a tuple
      • tuples may also be sliced
      • len() may also be used
      • Joining tuples: tuple 3 = tuple 1 + tuple 2
      • being immutable, tuples are faster than lists
    • Dictionaries
      • declared within { with each key / value pair separated by :
      • unordered
      • requires unique keys but values may be replicated
        • non-unique keys will not throw an error but will overwrite previous value
      • accessed with dictionary[key]
      • can be altered with dictionary[key] = value
      • new key - pairs can be added as per above
      • can use pop() and del()
    • Sets
      • unordered, unindexed group of elements that can have elements added or deleted but not edited
      • created by adding elements, separated by commas, within curly braces
      • set = {a, b, c, d}
      • add() to add a single element
      • update([,]) to add multiple elements
      • remove(element)
        • throws an error if element does not exist
      • discard(element)
        • no error if element does not exist
    • Boolean Operators:
      • same as java but also has
        • is
        • is not
      • and will return ‘true’ for any non-empty values, false for empty, 0, “”
